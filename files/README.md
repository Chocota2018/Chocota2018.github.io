# 📹 CAMVIEW — カメラ一括監視システム

WebRTC + WebSocket を使ったリアルタイムカメラ監視システムです。  
複数のクライアントのカメラ映像をホスト画面で一括管理できます。

## 動作の仕組み

```
[クライアントブラウザ] ──WebRTC P2P──► [ホストブラウザ]
         │                                      │
         └──────── WebSocket (シグナリング) ─────┘
                          │
                    [Node.js サーバー]
```

- **映像はサーバーを経由しません**（P2Pで直接送信）
- サーバーはWebRTCの接続確立（シグナリング）のみを担当
- `ws` パッケージのみ依存（軽量）

## セットアップ

```bash
# 依存パッケージのインストール
npm install

# サーバー起動
npm start
```

## アクセス

| 画面 | URL | 説明 |
|------|-----|------|
| ホスト | http://localhost:3000/host | 全カメラを一括表示 |
| クライアント | http://localhost:3000/client | カメラを送信する端末 |

## 使い方

1. **サーバーを起動** → `npm start`
2. **ホスト画面を開く** → ブラウザで `http://localhost:3000/host`
3. **クライアントを接続** → 別の端末・タブで `http://localhost:3000/client`
   - デバイス名を入力（任意）
   - カメラを選択
   - 「CONNECT & STREAM」をクリック
4. ホスト画面に自動的にカメラ映像が表示されます

## 機能

- **自動検出**: クライアントが接続すると即座にホスト画面に表示
- **レイアウト切替**: 1×1 / 2×2 / 3×3 / 4×4 / 自動
- **全画面表示**: 各カメラカードを全画面表示可能
- **自動再接続**: クライアントが切断後、自動で再接続
- **多人数対応**: 同時に複数クライアントをサポート
- **LAN対応**: 同一ネットワーク内の端末から接続可能

## LAN上の別端末から接続する場合

サーバーのIPアドレスを確認して：

```
http://<サーバーのIPアドレス>:3000/client
```

> **注意**: HTTPSが必要な場合（カメラアクセス制限）は、ngrok等でトンネルするか、  
> 自己署名証明書を使ったHTTPSサーバーに変更してください。
